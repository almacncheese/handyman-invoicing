# HandyQuote platform audit — 2026-07-15

**Scope:** Full security + correctness audit of HandyQuote (`handyman-invoicing` / quickhandyquote.com), with emphasis on the four per-tenant payment processors (Authorize.net, Stripe, Square, PayPal), multi-tenant isolation, and lifecycle/money paths.

**Method:** Source review of all API routes, payment libs, settings UI, public estimate pay flow; sibling-writer sweep; pre-live gate checklist; full Vitest suite + typecheck. Live sandbox charges not run (no contractor sandbox keys / `ENCRYPTION_KEY` not assumed present in local env).

**Verdict:** **CONDITIONAL GO** for product use with manual payments + e-sign. **NO-GO for real card charges in production** until Coolify `ENCRYPTION_KEY` is set, at least one processor sandbox E2E is green per provider, and this train’s fixes are deployed. Pro Stripe subscription remains blocked on Al’s platform Stripe keys.

## Per-tenant processors — can contractors set them up themselves?

| Processor | Settings UI | Secret storage | Contractor charge | Public customer pay | Notes |
|-----------|-------------|----------------|-------------------|---------------------|-------|
| **Authorize.net** | Yes — API Login ID, Client Key, Transaction Key, sandbox toggle | AES-256-GCM `secretEnc` | `POST /api/payments/charge` | `…/pay` | One-shot Accept.js |
| **Stripe** | Yes — publishable + secret key | same | intent + confirm | public intent + confirm | **Not** Connect; paste own keys |
| **Square** | Yes — Application ID, Location ID, Access token | same | charge | pay | Web Payments SDK |
| **PayPal** | Yes — Client ID + secret | same | intent + capture | public intent + capture | Button/popup (not Advanced Cards) |

**Owner-only write** on `PUT /api/business/payment-gateway`; staff read-only in Settings. Secrets never round-trip to the browser. Switching provider requires a new secret.

**Still required before real charges:**

1. Coolify **`ENCRYPTION_KEY`** (32-byte base64) — without it, saving gateway settings fails closed.
2. Contractor enters **their own** sandbox/live credentials in Settings → Payment processor.
3. One successful sandbox charge per processor (phone + public estimate).
4. HTTPS (prod has it; bare localhost tokenizers refuse except Stripe.js).

**Do not confuse** with HandyQuote’s own **$29/mo Pro** Stripe billing (`STRIPE_SECRET_KEY` / webhook / price) — that is Al’s platform account, not tenant deposit collection.

## Security strengths (preserve)

- Session JWT verified + **DB re-check** (active user/business); middleware cookie is UX only
- Cross-tenant reads use **404** via `assertSameBusiness`
- Public tokens validated shape before DB; pay amounts **server-derived** from token
- Manual + card payments: idempotency keys, claim-before-charge, invoice `FOR UPDATE` on settle
- Gateway secrets encrypted at rest; GET never returns secrets
- Confirm never trusts client-supplied Stripe PI / PayPal order id
- Seed wipe demo-scoped; full wipe dual-flag + refused in production
- Rate limits no longer trust spoofable `cf-connecting-ip` without Cloudflare
